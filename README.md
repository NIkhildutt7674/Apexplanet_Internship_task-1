# Data Cleaning and Preparation Project

## Project Overview

Data quality is the foundation of successful data analysis and machine learning projects. This project demonstrates a complete data cleaning and preprocessing workflow performed on a business dataset.

The objective was to identify and resolve common data quality issues, standardize data formats, engineer useful features, and produce an analysis-ready dataset.

---

## Objectives

* Understand the structure and business context of the dataset
* Create a comprehensive Data Dictionary
* Perform data profiling and quality assessment
* Identify missing values, duplicates, inconsistencies, and outliers
* Clean and transform the data
* Engineer new features for analysis
* Generate a final analysis-ready dataset

---

## Dataset Components

The dataset contains information related to customers, transactions, demographics, and business activities.

Example variables include:

* Customer ID
* Customer Name
* Date of Birth
* Transaction Date
* Gender
* Product Category
* Sales Amount
* Location

---

## Data Quality Issues Identified

### Missing Values

* Null values found in customer demographics and transaction attributes.
* Missing values handled using appropriate imputation techniques.

### Duplicate Records

* Duplicate customer and transaction records identified and removed.

### Inconsistent Formatting

* Date formats standardized.
* Text fields normalized.
* Category labels standardized.

### Outliers

* Extreme numerical values detected using statistical methods.
* Outliers reviewed and treated where appropriate.

---

## Data Transformations

### Date Standardization

Converted all date columns into a consistent datetime format.

### Free-Text Categorization

Standardized category names and customer attributes.

### Feature Engineering

Created new analytical features such as:

* Customer Age
* Customer Age Group
* Transaction Month
* Transaction Year
* Customer Tenure
